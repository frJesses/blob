    JS中的闭包如何理解？有什么作用？

    wwl 3/27/2023 Javascript

    闭包其实就是一个可以访问其他函数内部变量的函数。最简单的闭包就是在一个函数中，返回一个函数，创建的函数可以访问当前函数的变量

    闭包的本质：当前环境中存在父级环境中的引用

    因为通常情况下，函数内部的变量无法在外部进行访问（即全局变量与局部变量的区别），因此使用闭包的作用就是可以在外部访问某个函数内的变量，让这些变量的值存在内存中。

    function fun1() {
  var a = 1;
  return function () {
    console.log(a);
  };
}
fun1();
var result = fun1();
result(); // 1

    解释： 在控制台输出 1，因为函数 result 是一个闭包，result 函数具有 fun1 函数的作用域，在 result 中查找不到，就会到上一级作用域中查找，此时就是 fun1 函数中正好有个变量 a，即被输出。

    闭包有两个常用的用途

    • 可以在函数外部访问函数内部的变量
    • 将变量保存在内存中,避免多次使用闭包，防止内存溢出

    其实闭包的本质就是作用域链的一个特殊应用，只要了解了作用域链的创建过程，就能够理解闭包的原理

    let a = 1;
// fn 是闭包
function fn() {
  console.log(a);
}

function fn1() {
  let a = 1;
  // 这里也是闭包
  return () => {
    console.log(a);
  };
}
const fn2 = fn1();
fn2();

    闭包产生的原因：当我们访问一个变量的时候，会在当前的作用域中查找，如果没有找到，则会向上一级作用域继续查找，直到没有找到，这样的链路就是作用域链。

    var a = 1;
function fun1() {
  var a = 2;
  function fun2() {
    var a = 3;
    console.log(a); //3
  }
}

    • 那是不是只有返回函数才算是产生了闭包呢？其实也不是，回到闭包的本质，我们只需要让父级作用域的引用存在即可，因此还可以这么改代码

    var fun3;
function fun1() {
  var a = 2;
  fun3 = function () {
    console.log(a);

    因此最后返回的不管是不是一个函数，也不能说明没有产生闭包。

    闭包的表现形式

    1. 返回一个函数
    2. 在定时器、事件监听、Ajax 请求或者任何异步请求中，只要使用了回调函数，实际上就是在使用闭包。

setTimeout(function handler(){
  console.log('1');
}，1000);

    为什么上面的输出都是 6？

    • 因为 setTimeout 是宏任务，只有等主线程同步任务执行完成后才会执行。
    • 因为 setTimeout 也是一种闭包，在当前作用域中没有找到变量，此时变量 i 是全局变量，当 setTimeout 执行时，i = 6
